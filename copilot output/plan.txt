Le plan en détail — ce que tu fais exactement
H0–H1 : Setup & corpus
Tu télécharges les PDFs FAO/EPPO, tu les chunkes avec PyMuPDF, et tu les indexes avec paraphrase-multilingual-mpnet-base-v2 dans FAISS. Ce modèle multilingue est clé — il gère l'arabe et le français dans le même espace vectoriel, donc les questions en darija matchent les chunks en français.
H1–H3 : CNN olive disease
MobileNetV3 fine-tuné sur PlantVillage (subset olivier). Tu vises 3 classes minimum : œil de paon, anthracnose, verticilliose. Data augmentation agressive (blur, rotation, crop aléatoire) parce que les vraies photos terrain sont floues. Export TFLite pour tourner côté serveur léger.
H3–H5 : Le cœur anti-hallucination
C'est là que tu gagnes ou perds 30 points. Le système prompt Claude doit être bétonné : "Tu n'as le droit d'utiliser QUE le contexte ci-dessous. Si le score de similarité est < 0.75, réponds uniquement : ما عنديش معلومة كافية." Aucune complétion libre autorisée.
H5–H7.5 : Pipeline voix + PWA
Whisper-large pour la transcription (gère le dialecte tunisien), edge-tts pour la synthèse arabe sans clé API. PWA déployée sur Vercel avec Camera API. UI minimaliste : bouton photo + bouton micro + zone réponse.
H7.5–H9 : Durcissement démo
Tu simules exactement le scénario jury : photo de feuille malade → question vocale → réponse citée → question piège hors corpus → refus poli. Tu mesures la latence sur chaque étape. Tu prépares un fallback offline au cas où le WiFi lâche.
